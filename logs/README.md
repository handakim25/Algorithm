## 📚 학습 기록 가이드

### 왜 기록이 중요한가?
1. **진행 상황 가시화**: 얼마나 성장했는지 한눈에 확인
2. **복습 스케줄 관리**: 틀린 문제, 재풀이 필요 문제 추적
3. **패턴 인식**: 자주 틀리는 유형 파악 및 보완
4. **동기 부여**: 꾸준한 학습 습관 유지

### 기록 원칙
- **간결하게**: 너무 길게 쓰지 말고 핵심만
- **즉시 기록**: 문제 풀이 직후 바로 작성
- **정직하게**: 어려웠던 점, 틀린 이유를 솔직히 기록
- **행동 가능하게**: 막연한 반성보다 구체적인 개선 계획

## 💡 학습 기록 팁

### GitHub로 진행상황 시각화
```bash
# 매일 커밋하면 GitHub 잔디가 채워짐!
git add logs/daily/YYYY-MM-DD.md
git commit -m "📅 YYYY-MM-DD 학습 기록"
git push
```

### 효과적인 복습 주기 (에빙하우스 망각 곡선)
- **당일**: 문제 풀이 후 접근 방법 정리
- **1일 후**: 핵심 아이디어 재확인
- **7일 후**: 문제 다시 풀어보기
- **30일 후**: 유사 문제로 응용 확인
