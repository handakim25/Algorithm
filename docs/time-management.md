# ⏱️ 알고리즘 문제 풀이 시간 관리 전략

> 학습자가 문제 풀이 시 적절한 시간 투자로 최대의 학습 효과를 얻을 수 있도록 돕기 위한 가이드입니다.

---

## 📌 "30분 룰"의 정확한 의미

**30분은 절대적인 기준이 아니라 가이드라인입니다!** 🎯

- 30분은 "포기 시점"이 아니라 **체크포인트**입니다
- 난이도별로 다른 시간 기준을 적용합니다
- 학습 단계와 상황에 따라 유연하게 적용하세요

> 💬 **핵심 메시지**: "30분은 체크포인트이지, 포기 시점이 아닙니다!"

---

## 📊 난이도별 상세 시간 전략

### 🟢 프로그래머스 Level 1-2 (입문~초급)

| 시점 | 행동 |
|------|------|
| 0~20분 | 문제 이해 및 첫 접근법 시도 |
| 20~30분 | 막혔다면 문제 이해를 다시 하거나 작은 힌트 확인 |
| 45분 후 | 솔루션의 접근법만 확인 |
| 1시간 후 | 전체 솔루션 학습 후 재구현 |

---

### 🟡 프로그래머스 Level 3 (중급 — 현재 목표 수준)

| 시점 | 상태 | 행동 |
|------|------|------|
| 0~30분 | — | 집중 고민: 알고리즘 파악 + 손코딩 |
| **30분 시점** ✅ | 조금이라도 진전 있다면 | 15~30분 더 시도 |
| **30분 시점** 🚫 | 완전히 막혔다면 | 작은 힌트 확인 (어떤 알고리즘인지만) |
| **1시간 시점** ✅ | 부분 구현 성공 | 계속 진행 |
| **1시간 시점** 🚫 | 여전히 막힘 | 솔루션의 접근법 확인 |
| 1시간 30분 후 | — | 전체 솔루션 학습 후 재구현 |

---

### 🔴 프로그래머스 Level 4-5 / 백준 플래티넘 (고급)

| 시점 | 행동 |
|------|------|
| 0~1시간 | 충분히 고민, 다양한 접근법 시도 |
| 1시간 시점 | 진전 체크, 필요시 작은 힌트 확인 |
| 2시간 시점 | 접근법 확인 |
| 3시간 후 | 전체 솔루션 학습 후 재구현 |

---

## 🔍 진전 상황 판단 기준

### 🚫 완전히 막힌 상태 → 힌트 확인 필요

- 어떤 자료구조/알고리즘을 써야 할지 감도 안 잡힘
- 문제를 작은 단위로 쪼개지 못하겠음
- 10분 이상 아무 코드도 작성하지 못함
- 같은 생각만 반복하고 새로운 접근법이 떠오르지 않음

### ✅ 진전이 있는 상태 → 계속 진행

- 접근법은 생각났는데 구현이 어려움
- 일부 테스트케이스는 통과
- 새로운 아이디어가 계속 떠오름
- 디버깅 중이거나 최적화 시도 중

---

## 💡 힌트 확인 단계별 전략

### 1단계 — 가벼운 힌트 (막혔을 때, 5분 고민 후)

- 🏷️ 태그/분류만 확인 (어떤 알고리즘 카테고리인지)
- 📝 입출력 예제 하나 더 만들어보기
- 📖 문제를 다시 천천히 읽기

### 2단계 — 중간 힌트 (30분 후)

- 🔑 사용해야 하는 핵심 알고리즘/자료구조 확인
- ⏳ 시간복잡도 제한 확인
- 🔎 비슷한 유형 문제 검색

### 3단계 — 큰 힌트 (1시간 후)

- 📋 솔루션의 전체 접근법 (의사코드 수준)
- 💡 핵심 아이디어만 확인하고 직접 구현

### 4단계 — 전체 솔루션 (1.5~2시간 후)

- 📄 전체 코드 확인
- 🧠 왜 그렇게 풀었는지 이해
- 🔄 다음날 다시 스스로 풀어보기

---

## 🎓 학습 효과를 높이는 방법

### 힌트를 본 후에도 학습 효과 유지하기

1. 힌트만 보고 바로 창 닫기
2. 힌트 기반으로 스스로 구현
3. 솔루션을 봤다면:
   - ✅ 코드를 **타이핑**하며 이해 (복붙 금지 ❌)
   - ✅ 주석으로 각 부분 설명 달기
   - ✅ 다음날 반드시 **백지 상태에서 재풀이**
   - ✅ 1주일 후 다시 풀기

### 시간 투자 vs 학습 효율

| 고민 시간 | 효과 |
|-----------|------|
| 너무 빨리 포기 (10분 이내) | 사고력 향상 안 됨 ❌ |
| **적절한 고민 (30~60분)** | **최적의 학습 효과** ✅ |
| 과도한 고민 (3시간 이상) | 효율 떨어짐, 번아웃 위험 ⚠️ |

---

## 🗓️ 상황별 추천 전략

### ☀️ 시간이 충분할 때 (주말, 여유있는 날)

- 평소보다 30분~1시간 더 고민
- 여러 접근법 시도
- 브루트포스부터 최적화까지 단계적 구현

### 🏃 시간이 부족할 때 (평일, 바쁜 날)

- 20~30분 집중 시도
- 빠르게 힌트 확인 후 학습
- 다음날 복습 계획 세우기

### 🆕 새로운 유형을 처음 만났을 때

- 기본 개념부터 학습
- 쉬운 예제 문제부터 시작
- 힌트를 적극 활용해도 OK 👍

### 🔁 익숙한 유형일 때

- 평소보다 긴 시간 도전
- 최적화에 집중
- 다양한 풀이법 시도

## 🏆 실전 코딩테스트 대비

### 연습 단계 vs 실전 대비

| 단계 | 전략 |
|------|------|
| **학습 단계** | 시간 넉넉히, 깊게 이해 |
| **실전 대비** (시험 1~2달 전) | 시간 제한 엄격히 (30분/문제) |

### 모의 테스트 시

- ⏰ 시간 제한 엄수
- 📝 못 푼 문제는 메모만 하고 넘어가기
- 📚 테스트 후 복습 시간 별도 확보
